# 新闻综述聚合设计

**日期:** 2026-02-24
**状态:** 已批准

## 问题

去重后的新闻中，仍存在大量"不同角度报道同一事件"的情况。当前邮件中"重点新闻"+"其他新闻"的扁平列表无法体现事件关联性，读者需要自行识别哪些新闻在讲同一件事。

## 目标

将邮件中的新闻板块从扁平列表改为**综述组 (news_clusters)** 格式：
- 每个综述组 = 综述标题 + 整合摘要 + impact_tag + 原始新闻链接列表
- 综述组之间按重要性排序（最重要的排第一）
- 每个综述组内的原始新闻也按重要性排序
- 原始新闻标题可点击跳转到原文
- 单条独立新闻也形成一个综述组，保持格式统一

## 方案

### 改动范围

仅修改 `utils/digest_controller.py`，不改动 `main.py` 管道流程。

### 1. Python 预处理：标题去重过滤

在 `build_llm_prompt()` 构建新闻列表之前，新增 `_pre_deduplicate_news()` 方法：

- **目的**：剔除高度重复的新闻条目（同一新闻的不同来源完全相同标题），减少传给 LLM 的冗余
- **逻辑**：复用现有 `Deduplicator` 中的标题归一化思路（小写化 + 分词 + 词形还原），计算 Jaccard 相似度
- **阈值**：≥ 0.75 视为重复，保留 relevance_score 更高 或 时间更新 的一条
- **注意**：这不是语义聚合，只是精确/近似去重，语义聚合完全交给 LLM

### 2. LLM Prompt 改造

修改 `build_llm_prompt()` 中的【你的任务】部分：

- 删除原有的 `key_news`（恰好5条）+ `other_news`（最多5条）指令
- 新增 `news_clusters` 指令：
  - 将所有新闻按事件/主题聚合为综述组
  - 报道同一事件的不同角度新闻合并到同一个 cluster
  - 独立新闻单独成组
  - cluster 之间按重要性排序（第一个最重要）
  - 每个 cluster 内 sources 按重要性排序
  - 每个 cluster 必须标注 impact_tag（对贵金属整体影响方向）
  - 综述标题和摘要使用中文，整合核心含义

### 3. JSON Schema 变更

```
原来: { subject, key_news[5], other_news[≤5], analysis }
改为: { subject, news_clusters[≥1], analysis }
```

每个 cluster 结构：
```json
{
  "cluster_title": "综述标题（中文）",
  "cluster_summary": "整合摘要，1-3句话概括核心含义（中文）",
  "impact_tag": "Bullish | Bearish | Neutral",
  "sources": [
    {
      "title": "原始新闻标题",
      "source": "来源名称",
      "url": "原文链接",
      "timestamp": "HH:MM 或空字符串"
    }
  ]
}
```

### 4. HTML 渲染变更

原来的两个板块（"重点新闻" + "其他新闻"）→ 合并为一个 **"新闻综述"** 板块。

每个 cluster 渲染为：
```
┌──────────────────────────────────┐
│ [利多] 综述标题                    │
│ 综述摘要文字...                    │
│                                  │
│  · 原始标题1 (来源 · 时间)  ← 可点击 │
│  · 原始标题2 (来源 · 时间)  ← 可点击 │
│  · 原始标题3 (来源 · 时间)  ← 可点击 │
└──────────────────────────────────┘
```

- 保持现有的 impact_tag 配色方案（利多绿 / 利空红 / 中性灰）
- cluster 之间用分隔线区分

### 5. 不改动的部分

- `main.py` 管道流程不变
- `analysis` 板块保持不变
- 市场行情、经济指标、COMEX 板块全部不变
- `plain_text` 模式不受影响
- 邮件模板整体结构不变（header、footer、其他 section）
